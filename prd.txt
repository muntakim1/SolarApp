1. Executive Summary
SolventZ is a comprehensive solar solutions platform comprising a mobile application (Android and iOS) built with React Native and an admin web panel developed using Next.js. The platform is engineered to digitize the complete solar product sales lifecycle — from customer onboarding and product discovery through purchase, project management, and post-installation support.

The application addresses a growing demand for accessible, streamlined solar energy procurement in markets where solar adoption is accelerating. SolventZ will enable customers to browse and purchase solar panels, inverters, batteries, and VED (Voltage Enhancement Devices), request on-site surveys, receive custom project quotations, track orders, and submit after-sales support tickets — all from a single, branded mobile interface.

The admin web panel provides operations teams with full control over product catalog management, order processing, customer query resolution, and survey request handling, backed by real-time data from a Supabase-powered database.

1.1 Business Objectives
Digitize the end-to-end solar sales process, reducing manual effort by at least 60%
Improve customer acquisition and retention through a seamless mobile experience
Provide operations teams with a centralized admin hub for order and inventory management
Enable scalable project management from survey request to installation confirmation
Establish a foundation for AI-powered features and automation in future product iterations

3.2 Authentication Module
3.2.1 Registration Flow
New users register with their email address and receive an OTP for verification.
Step
Details
1. Email Entry
User enters email address. Frontend validates format in real time.
2. OTP Request
Supabase Auth sends a 6-digit OTP to the provided email. OTP expires in 10 minutes.
3. OTP Entry
User enters the 6-digit code. Three failed attempts lock the account for 15 minutes.
4. Profile Setup
User enters: Full Name, Phone Number, City/Region. All fields required.
5. Session Created
Supabase returns JWT token stored securely in device keychain / SecureStore.

3.2.2 Login Flow
User enters registered email address
Supabase sends OTP to email
User enters OTP — authenticated on success
Failed login after 5 attempts triggers a 30-minute lockout and admin notification

3.2.3 Forgot Password / Reset
User enters email on the Forgot Password screen
Supabase triggers password reset email with a secure link (expires in 1 hour)
Link opens the app via deep link to a new password creation screen
Password must be 8+ characters, contain at least one number and one special character

3.2.4 Logout
Accessible from the profile screen and main navigation drawer
Clears JWT token from device storage
Invalidates Supabase session
Redirects to the unauthenticated home/login screen

3.2.5 Error States
Invalid email format — inline validation error
Unregistered email on login — prompt to register
Expired OTP — 'Resend OTP' button with 60-second cooldown
Network timeout — retry prompt with error toast

5. Authentication System
5.1 Supabase Auth Configuration
Email OTP authentication for mobile app customers
Email + Password authentication for admin panel users
JWT tokens with a 1-hour expiry and automatic refresh
Session stored in Supabase Auth — all sessions invalidated on password change
Row Level Security (RLS) enforced at the database level based on authenticated user identity

5.2 Mobile App Session Management
Scenario
Behavior
App Launch — Logged In
JWT validated. If valid, navigate to home. If expired, attempt silent refresh.
Silent Refresh Fails
Navigate to login screen. Show 'Session expired' toast.
Manual Logout
Call Supabase signOut(). Clear local storage. Navigate to login.
Concurrent Session
Only one active session per user on mobile. New login invalidates previous.
Password Reset
All existing sessions invalidated on successful password reset.

5.3 Admin Panel Security
HTTPS enforced on all admin panel routes
CSP headers configured on Next.js middleware
Rate limiting: max 10 failed login attempts per IP per hour
Audit log: every admin action recorded with user ID, action type, timestamp, and affected record ID
2FA (TOTP) available for Super Admin accounts (Phase 2)

6.3 Indexing Strategy
products: INDEX on category_id, brand, is_active, is_deleted
orders: INDEX on user_id, status, created_at
order_items: INDEX on order_id, product_id
surveys: INDEX on user_id, status, created_at
support_tickets: INDEX on user_id, status, assigned_to, created_at

6.4 Row Level Security Policies
users: Users can read and update only their own record. Admins can read all.
products: All authenticated users can read active, non-deleted products. Only admins can insert, update, delete.
orders: Users can read only their own orders. Admins can read and update all.
order_items: Users can read only items belonging to their orders. Admins can read all.
surveys: Users can read and insert their own surveys. Admins can read and update all.
support_tickets: Users can read and insert their own tickets. Assigned agents and admins can read and update all.
